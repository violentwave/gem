# M1 — Weekly Health / Drift Operating Cycle

## Purpose

Run weekly health and drift checks manually. No timers. No automation.

## Boundaries

- Do not enable timers.
- Do not modify system configs.
- Do not expose services.
- Do not install packages.
- Do not use sudo.

## Steps

1. Run `gemma-monitor-daily`
2. Run `gemma-monitor-drift`
3. Review output for any FAIL or unexpected WARN
4. If FAIL found:
   - Identify failing check
   - Consult TROUBLESHOOTING.md
   - Fix or escalate
5. If drift detected:
   - Check if expected (new helpers, updated docs)
   - If unexpected: investigate and document
6. Update Notion tracker with results
7. Update CURRENT_STATE.md if significant changes

## Expected Output

- Monitor results: 11/11 PASS (daily), variable (drift)
- Notion tracker updated
- Log files in `~/.local/state/bazzite-security/logs/`

## Report Format

```markdown
## M1 Weekly Check — YYYY-MM-DD

### Daily Monitor
- Status: PASS/WARN/FAIL
- Details: [paste output]

### Drift Monitor
- Status: PASS/WARN/FAIL
- Details: [paste output]

### Actions Taken
- [list any fixes or investigations]

### Next Check
- Date: [next Monday]
```

## Stop Conditions

Stop and escalate if:
- Any FAIL cannot be resolved
- Drift indicates security issue
- System becomes unstable
