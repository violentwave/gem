Prompt: Phase 12B1 — Agent Zero Read-Only Runtime Dry-Run

Purpose:
Run an explicit, human-approved Agent Zero read-only runtime dry-run. Agent Zero may be started ONLY if the human explicitly approves in this prompt. It must use a single approved read-only source, produce one structured briefing, stop Agent Zero afterward, and not persist memory, modify config, grant write access, run OpenCode, use sudo, mutate memory, or enable learning/training.

Prerequisites (ALL must be true):
- [ ] Human has read docs/phase12/PHASE12B_AGENT_ZERO_READONLY_BRIDGE_REVIEW.md
- [ ] Human has read docs/phase12/AGENT_ZERO_SPACE_AGENT_BRIDGE_BOUNDARIES.md
- [ ] Human has read docs/phase12/AGENT_ZERO_READONLY_BRIEFING_PACKET_TEMPLATE.md
- [ ] Human explicitly approves starting Agent Zero in this prompt
- [ ] Single read-only source document is identified
- [ ] Human is available to review output within 10 minutes
- [ ] Human understands stop conditions

Hard Boundaries:
- Start Agent Zero ONLY if human explicitly checks the approval box below.
- Use exactly ONE read-only source document.
- Do not modify Agent Zero config (~/.config/agent-zero/).
- Do not grant repo write authority.
- Do not grant host write authority.
- Do not enable autonomous remediation.
- Do not run OpenCode prompts autonomously.
- Do not ingest, index, mutate, or promote memory.
- Do not enable learning/training.
- Do not use sudo.
- Do not modify system/security configs.
- Do not allow Agent Zero to make network requests to external services.

Runtime Workflow:
1. If approved, start Agent Zero with `agent-zero-up`.
2. Wait for container to be ready (port 5080).
3. Provide the single approved read-only source to Agent Zero.
4. Request a structured Markdown briefing using the template in AGENT_ZERO_READONLY_BRIEFING_PACKET_TEMPLATE.md.
5. Capture the briefing output.
6. Stop Agent Zero with `agent-zero-down`.
7. Confirm container is stopped.
8. Human reviews briefing using the review checklist.
9. Document results in a new file under ~/offload/security-reports/manual/.

Stop Conditions (stop immediately and run agent-zero-down):
- Agent Zero requests file write access.
- Agent Zero requests system commands.
- Agent Zero requests network access.
- Agent Zero requests memory mutation.
- Agent Zero requests learning/training execution.
- Agent Zero attempts to run OpenCode prompts.
- Output contains suspicious or out-of-bounds recommendations.
- Human reviewer is not available.
- Dry-run exceeds 10-minute timebox.

Expected Output:
- One structured Markdown briefing
- Boundary compliance table
- Human review section with reviewer signature
- Explicit "no execution authorized" section
- Report saved to ~/offload/security-reports/manual/

Human Approval:
To proceed, the human must explicitly state: "I approve starting Agent Zero for a read-only dry-run on [source document]."

Note:
This prompt requires explicit human authorization. Do not start Agent Zero without the explicit approval statement above.
