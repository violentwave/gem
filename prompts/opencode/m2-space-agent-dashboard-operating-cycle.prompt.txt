# M2 — Space Agent Dashboard Operating Cycle

## Purpose

Generate dashboard packets for manual review in Space Agent.

## Boundaries

- Space Agent remains manual UI only.
- No always-on dashboard.
- No separate dashboard service.
- No LAN exposure.

## Steps

1. Generate dashboard packet:
   ```bash
   gemma-monitor-daily > ~/offload/security-reports/dashboard-packets/packet-$(date +%Y-%m-%d).md
   gemma-monitor-drift >> ~/offload/security-reports/dashboard-packets/packet-$(date +%Y-%m-%d).md
   ```
2. Launch Space Agent:
   ```bash
   ~/Applications/Space-Agent.AppImage
   ```
3. Configure Gemma provider (if needed):
   - Settings → AI Providers → Local
   - URL: http://127.0.0.1:11434
   - Model: gemma4-e4b-bazzite:latest
4. Paste packet into Space Agent chat
5. Ask Gemma to summarize status
6. Review insights
7. Document any action items
8. Close Space Agent when done

## Expected Output

- Dashboard packet file
- Gemma summary
- Action items (if any)

## Report Format

```markdown
## M2 Dashboard Review — YYYY-MM-DD

### Packet
- File: [path]
- Size: [lines]

### Gemma Summary
- [paste summary]

### Action Items
- [list]

### Next Review
- Date: [next week]
```

## Stop Conditions

Stop if:
- Space Agent cannot connect to Ollama
- Gemma produces incorrect summaries
- Any security concerns raised
