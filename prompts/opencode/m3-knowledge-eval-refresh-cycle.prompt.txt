# M3 — Knowledge + Eval Refresh Cycle

## Purpose

Refresh knowledge pack and evals monthly. No ingestion. No indexing unless docs changed.

## Boundaries

- Do not run ingestion.
- Do not mutate RuVector.
- Do not train or fine-tune.
- Do not enable autonomous learning.

## Steps

1. Review knowledge pack docs for staleness:
   ```bash
   ls -la ~/.local/share/bazzite-security/gemma-knowledge/docs/
   ```
2. Check doc ages:
   ```bash
   find ~/.local/share/bazzite-security/gemma-knowledge/docs/ -type f -mtime +30
   ```
3. If docs need updating:
   - Edit source docs in `~/.config/bazzite-security/`
   - Copy updated docs to knowledge pack
   - Re-index:
     ```bash
     gemma-knowledge-index
     gemma-knowledge-crossref
     ```
4. Run eval validators:
   ```bash
   gemma-evals-check
   gemma-examples-check
   gemma-evals-status
   ```
5. If evals FAIL:
   - Identify failing cases
   - Fix knowledge docs or eval cases
   - Re-run until PASS
6. Document refresh in CURRENT_STATE.md
7. Update Notion tracker

## Expected Output

- Updated knowledge pack (if changes made)
- Re-indexed chunks (if docs changed)
- All evals PASS

## Report Format

```markdown
## M3 Knowledge + Eval Refresh — YYYY-MM-DD

### Docs Reviewed
- [list docs and ages]

### Changes Made
- [list changes or "none"]

### Index Status
- Chunks: [count]
- Crossrefs: [count]

### Eval Results
- gemma-evals-check: PASS/FAIL
- gemma-examples-check: PASS/FAIL

### Next Refresh
- Date: [next month]
```

## Stop Conditions

Stop if:
- Eval failures cannot be resolved
- Knowledge pack corruption detected
- Index rebuild fails
