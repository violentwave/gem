# M8 — External Dataset Vetting (Hugging Face Review)

**Phase:** M8 — External Dataset Vetting  
**Type:** Review-only maintenance phase  
**Status:** PLANNED (not yet executed)

---

## Purpose

Review specific Hugging Face dataset candidates for potential future example curation, without downloading datasets by default, performing training, or bulk-importing data.

This phase produces **candidate packets** for human review. No dataset is downloaded, no model is trained, no examples are added without explicit human approval.

---

## Hard Boundaries

- **NO dataset download by default** — Datasets are reviewed via metadata and sample inspection only.
- **NO training** — No fine-tuning, LoRA, QLoRA, or SFT.
- **NO bulk import** — Individual samples may be reviewed; bulk ingestion is prohibited.
- **NO automated ingestion** — No scripts that automatically add dataset content to the knowledge pack or example set.
- **NO secrets** — Do not print Notion tokens, API keys, or secrets.
- **NO system changes** — Do not install packages, modify configs, or expose services.

---

## Prerequisites

Before running M8:

1. M7 (Training Readiness Review) must be complete.
2. [HUGGINGFACE_DATASET_VETTING_POLICY.md](../maintenance/HUGGINGFACE_DATASET_VETTING_POLICY.md) must be reviewed.
3. Specific dataset candidates must be identified (or a search query provided).

---

## Workflow

### Step 1: Identify Candidates

Identify 3–5 Hugging Face datasets matching the Bazzite domain:

- Linux system administration
- Security best practices
- Open-source tooling documentation
- Technical writing / documentation

Search queries (do not download):
- `linux administration instructions`
- `security best practices qa`
- `system administration dataset`

### Step 2: License Review

For each candidate dataset:

- [ ] Read dataset card (HF page metadata)
- [ ] Identify license (MIT, Apache-2.0, CC, etc.)
- [ ] Confirm commercial/personal use allowed
- [ ] Confirm derivative works allowed
- [ ] Document license in candidate packet

**Reject if:** License is unknown, restrictive (NC, ND), or incompatible.

### Step 3: Sample Review

For each candidate dataset:

- [ ] Inspect 5–10 sample rows (via HF dataset viewer or sample download)
- [ ] Assess quality (grammar, coherence, accuracy)
- [ ] Assess Bazzite domain alignment
- [ ] Document findings in candidate packet

**Reject if:** Samples are low quality, off-topic, or machine-generated garbage.

### Step 4: Contradiction Scan

For each sample reviewed:

- [ ] Scan for Bazzite contradictions (apt vs rpm-ostree, ufw vs firewalld, etc.)
- [ ] Scan for denied patterns (secrets, PII, private code, raw logs)
- [ ] Document contradictions and denied patterns found

**Reject if:** Unresolvable contradictions or denied patterns found.

### Step 5: Candidate Packet Output

For each dataset, produce a candidate packet:

```markdown
## Dataset Candidate: {name}

- **HF URL:** {url}
- **License:** {license}
- **Size:** {rows} rows
- **Domain:** {category}
- **Quality:** {high/medium/low}
- **Bazzite Alignment:** {high/medium/low}
- **Contradictions Found:** {list or "none"}
- **Denied Patterns Found:** {list or "none"}
- **Samples Reviewed:** {count}
- **Rewritable Samples:** {count}
- **Recommendation:** {accept/reject/review}
- **Reviewer:** {human name}
- **Date:** {YYYY-MM-DD}
```

### Step 6: Human Approval

- [ ] Submit all candidate packets to human reviewer
- [ ] Obtain explicit approval before any dataset download or example creation
- [ ] Record approval decision
