Prompt: Phase 12J1 — Notion Update Packet Review

Purpose:
Produce reviewed Notion update packets from repo state. This prompt does NOT directly update Notion. It generates proposed updates for human/ChatGPT review.

Context:
- Notion database tracks phases, status, commits, blockers
- Repo docs (CURRENT_STATE.md, ROADMAP.md) are source of truth
- OpenCode reads repo docs and generates update proposals
- Human reviews and approves before applying to Notion

Prerequisites (ALL must be true):
- [ ] Human has read docs/phase12/NOTION_OPENCODE_SYNC_POLICY.md
- [ ] Human understands that OpenCode does NOT write to Notion directly
- [ ] Human explicitly approves generating update packets
- [ ] Repo docs are current and committed

Hard Boundaries:
- Do not write to Notion directly.
- Do not store Notion tokens in repo.
- Do not print Notion secrets.
- Do not modify Notion schema.
- Do not mark future phases as DONE early.
- Do not bypass human review.

Allowed:
- Read repo docs
- Compare repo state to last known Notion snapshot
- Generate update packet (Markdown/JSON)
- Propose field updates
- Document drift
- Save packet to docs/phase12/notion-update-packets/ or ~/.cache/

Workflow:

Step 1 — Read repo source of truth:
- docs/live-system/CURRENT_STATE.md
- docs/roadmap/ROADMAP.md
- docs/phase12/PHASE12H_BRIDGE_READINESS_CLOSEOUT.md
- Closeout docs for completed phases

Step 2 — Read last Notion snapshot (if available):
- ~/.cache/bazzite-security/notion/latest-snapshot.json
- Or user-provided exported data

Step 3 — Compare and identify drift:
- Phase statuses
- Commit SHAs
- Blocker text
- Next actions
- Approval states

Step 4 — Generate update packet:

```markdown
# Notion Update Packet — YYYY-MM-DD

## Source of Truth
- Repo commit: <SHA>
- Snapshot date: <date>

## Proposed Changes

### Phase X
- Field: Status
- Current (Notion): IN_PROGRESS
- Proposed: COMPLETE
- Reason: Closeout doc committed, validators passed
- Evidence: docs/phase12/PHASE12X_CLOSEOUT.md

### Phase Y
- Field: Current Blocker
- Current (Notion): (empty)
- Proposed: Space Agent not installed
- Reason: Phase 12C1 blocked pending installation
- Evidence: docs/phase12/PHASE12C_SPACE_AGENT_MANUAL_UI_BRIDGE_REVIEW.md

## Validation Checklist
- [ ] Closeout doc exists
- [ ] Commit SHA matches
- [ ] Validators passed
- [ ] No secrets exposed

## Human Review
Reviewer: ________________
Approved: [ ] YES  [ ] NO  [ ] DEFER
Date: ________________
Notes: ________________
```

Step 5 — Save packet:
- Path: docs/phase12/notion-update-packets/YYYY-MM-DD-update-packet.md
- Or: ~/.cache/bazzite-security/notion/update-packets/YYYY-MM-DD-update-packet.md

Step 6 — Human applies to Notion:
- Human reviews packet
- If approved, human updates Notion (manually or via ChatGPT connector)
- If rejected, human documents reason

Stop Conditions:
- Packet proposes marking future phase as DONE early
- Packet proposes updating schema
- Packet contains secrets
- Packet is generated without reading source of truth

Expected Output:
- Update packet file
- Drift report
- Human review section

Note:
This prompt generates PROPOSALS only. Actual Notion updates require explicit human action.
