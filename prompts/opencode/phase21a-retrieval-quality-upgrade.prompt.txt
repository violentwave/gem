# Phase 21A — Retrieval Quality Upgrade

**Context:** Phase 16B identified chunking improvements. Phase 21A implements them.

**Goal:** Improve Stage 3A and RuVector retrieval quality with better chunking.

**Tasks:**
1. Implement larger chunk size (~800 chars)
2. Increase overlap to 100 chars
3. Skip pure headers in indexing
4. Preserve code blocks as units
5. Merge short paragraphs
6. Add cross-reference metadata
7. Re-index knowledge pack
8. Compare retrieval quality vs baseline

**Boundaries:**
- No RuVector mutations without approval
- Stage 3A fallback preserved
- Validate before replacing index

**Depends on:** Phase 20 complete (knowledge docs added)

**Output:** Updated chunk index, quality comparison report
