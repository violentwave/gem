Prompt: Phase 12K — Space Agent Installation Readiness Assessment

Purpose:
Assess whether Space Agent can be safely installed on Bazzite/Fedora Atomic without system changes, sudo, or package manager modifications.

Context:
- Space Agent is currently NOT installed on the host
- Space Agent binary: NOT FOUND in PATH
- Space Agent container: NOT FOUND
- Space Agent config: NOT FOUND (~/conf/ does not exist)
- Space Agent data dir: EXISTS (~/.config/space-agent/ — Electron app data only)
- Host: Bazzite/Fedora Atomic Desktop
- Package manager: rpm-ostree (not apt)
- GUI preference: Flatpak
- CLI preference: Homebrew or user-local install
- Isolation preference: containers/distrobox/quadlet

Prerequisites (ALL must be true):
- [ ] Human has read docs/phase12/PHASE12J_OPENCODE_NOTION_READONLY_SYNC_DESIGN.md
- [ ] Human has read docs/phase12/PHASE12C_SPACE_AGENT_MANUAL_UI_BRIDGE_REVIEW.md
- [ ] Human explicitly approves running Phase 12K
- [ ] No installation will be performed in this phase

Hard Boundaries:
- Do NOT install Space Agent in this phase.
- Do NOT use sudo.
- Do NOT install packages with rpm-ostree, dnf, or apt.
- Do NOT modify system configs.
- Do NOT expose services on LAN.
- Do NOT start Space Agent.
- Do NOT modify firewall, USBGuard, ClamAV, Lynis, or systemd.

Allowed:
- Research official Space Agent sources
- Check GitHub releases page
- Check FlatHub for Flatpak
- Check if AppImage is available
- Document installation requirements
- Assess compatibility
- Create readiness report

Research Checklist:

1. Official Source
   - Space Agent GitHub repository URL
   - Latest release version
   - Release assets (AppImage, deb, rpm, etc.)
   - Installation instructions

2. Bazzite Compatibility
   - Does the official install method require sudo?
   - Does it require rpm-ostree layering?
   - Does it conflict with existing packages?
   - Does it require system-wide changes?

3. Alternative Install Methods
   - AppImage: Can it run user-local without sudo?
   - Flatpak: Is it on FlatHub?
   - Homebrew: Is there a formula?
   - Distrobox: Can it run in a container?
   - Manual extraction: Can binaries be extracted to ~/.local/bin/?

4. Risk Assessment
   - Does install require write to /usr or /opt?
   - Does it create systemd services?
   - Does it modify ~/.config/ in ways that conflict?
   - Does it require external API keys?

Deliverables:
- docs/phase12/PHASE12K_SPACE_AGENT_INSTALLATION_READINESS.md
- Installation readiness decision:
  - APPROVED_FOR_INSTALL: Safe method found, user can proceed with explicit approval
  - BLOCKED: No safe method found
  - NEEDS_REVIEW: Partial findings, needs human decision

Expected Output:
- Readiness report with findings
- Recommended install method (if any)
- Risk assessment
- Prerequisites for Phase 12L

Note:
This is an assessment phase only. No installation is performed.
