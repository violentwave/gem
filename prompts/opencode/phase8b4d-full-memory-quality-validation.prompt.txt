## Tasks

### 1. Run Full Gate 1 Validation (Revised Metric)

Using the new source-equivalence metric:

1. Run 10+ known eval queries through RuVector semantic prototype
2. Run same queries through Stage 3A
3. Classify each top source by family (Policy/Operational/Advisory/Stage3A)
4. Calculate:
   - Exact overlap: `(common_filenames / total_unique_filenames) * 100`
   - Equivalence: `(equivalent_pairs / total_pairs) * 100`
5. Determine source authority (which system is more authoritative)
6. Assign classification labels
7. Check for factual contradictions
8. Document FINAL_POLICY.md dominance explanations

**Pass Criteria (ANY of):**
- ≥70% exact overlap, OR
- ≥70% equivalence + no contradictions, OR
- semantic_more_authoritative explained, OR
- stage3a_more_authoritative explained

**Deliverable:** Revised comparison table with source families and equivalence scores

### 2. Run Gate 3 Semantic Quality (10+ Queries)

1. Run 10+ known eval questions through RuVector
2. Check relevance scores for top 5 results
3. Manual spot-check top results
4. Identify queries where semantic should excel

**Pass Criteria:**
- Relevant sources in top 5
- No hallucinations
- Precision ≥ Stage 3A on complex queries

### 3. Run Gate 4 Source Spot-Check (10 Queries)

1. Run 10 representative queries
2. Manually verify top 3 are relevant
3. Check for off-topic results

**Pass Criteria:**
- 8/10 queries return relevant in top 3
- No completely off-topic in top 3

### 4. Prepare Gate 5 (Optional - Do NOT Run Heavy Generation)

- Note: Gate 5 requires answer generation
- Do NOT run heavy generation unless explicitly approved
- If approved: Compare Gemma + RuVector vs Gemma + Stage 3A
