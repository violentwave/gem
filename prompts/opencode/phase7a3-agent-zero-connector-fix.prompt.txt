# Phase 7A.3: Agent Zero Connector Smoke Test

**Goal:** Run a harmless connector smoke test to verify the A0 CLI to Agent Zero integration is fully operational.

**When to run this prompt:**
- Only if Phase 7A.2 report says `connector_working`
- After A0 CLI is installed and connectivity is verified

---

## Hard Boundaries (Same as Phase 7A.2)

- Do not use sudo.
- Do not install packages system-wide.
- Do not use host networking.
- Do not inspect .env contents or secrets.
- Do not modify firewall, USBGuard, ClamAV, Lynis, rpm-ostree, systemd timers, Ollama config, or OpenCode permissions.
- Do not create system services or autostart entries.
- Do not leave Agent Zero running at the end unless explicitly documented.
- Keep generated reports in ~/projects/gem/docs or canonical paths.

---

## Allowed Actions

- Start Agent Zero with existing helper script
- Run harmless A0 CLI smoke tests that don't execute actual agent tasks
- Use `AGENT_ZERO_HOST` env var if needed
- Create connector verification report
- Stop Agent Zero at end unless intentionally left running

---

## Tasks

### 1. Start Agent Zero

```bash
agent-zero-up
sleep 8
agent-zero-status
```

### 2. Run Harmless Connector Smoke Test

**Option A - Version/Status check:**
```bash
AGENT_ZERO_HOST=http://127.0.0.1:5080 a0 --version
```

**Option B - Help with connection (doesn't spawn agents):**
```bash
AGENT_ZERO_HOST=http://127.0.0.1:5080 a0 --help
```

**Option C - If there's a documented ping/health command:**
```bash
# Use only if documented and non-mutating
AGENT_ZERO_HOST=http://127.0.0.1:5080 a0 <harmless-command>
```

**Do NOT:**
- Do NOT spawn actual agents
- Do NOT run tasks against the host
- Do NOT grant filesystem access
- Do NOT run computer-use operations

### 3. Document Smoke Test Results

If smoke test passes:
- Record successful connection
- Note any harmless output

If smoke test fails:
- Document error
- Identify next steps

### 4. Stop Agent Zero

```bash
agent-zero-down
agent-zero-status
```
