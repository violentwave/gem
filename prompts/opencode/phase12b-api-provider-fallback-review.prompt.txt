Prompt: Phase 12B-API — External API Provider Fallback Review (Optional)

Purpose:
Review whether to add an external LLM API provider (OpenRouter, OpenAI, Anthropic, etc.) as a fallback for Agent Zero when local Gemma/Ollama is unavailable or impractical.

Context:
- Local Gemma/Ollama works from Agent Zero container via 10.0.2.2:11434.
- First model load may take >30 seconds.
- External API would provide faster response times.
- External API requires API key and network access.

Hard Boundaries:
- Do NOT store API keys in the repo.
- Do NOT store API keys in tracked config files.
- Do NOT print API keys in output.
- Use user-local secrets only (e.g., ~/.local/share/agent-zero/usr/secrets.env).
- Use read -s or manual secret entry for key input.
- Verify .gitignore covers secrets files.
- Keep local Gemma/Ollama as the PRIMARY/PREFERRED path.
- External API is FALLBACK ONLY.

Security Requirements:
1. Secret Storage:
   - Store in ~/.local/share/agent-zero/usr/secrets.env
   - Never commit to git
   - Never paste into logs or reports

2. Provider Selection:
   - Prefer OpenRouter (supports multiple models, pay-per-use)
   - Alternative: OpenAI, Anthropic, Groq
   - Do not use personal/expensive tiers for testing

3. Network Safety:
   - Provider must use HTTPS
   - No plaintext API key transmission
   - Verify TLS certificate

4. Reversibility:
   - Must be able to remove provider config easily
   - Must be able to switch back to local Gemma

Review Questions:
- Is external API fallback necessary for current use case?
- Which provider offers best cost/performance for read-only briefings?
- Are there free-tier or low-cost options suitable for testing?
- What is the data privacy implication of sending prompts to external API?

Recommendation:
Only proceed with external API fallback if:
1. Local Gemma proves consistently too slow
2. User explicitly requests it
3. User provides their own API key
4. User understands data privacy implications

Note:
This is a planning-only prompt. Do not configure external API keys in this phase.
