Prompt: Phase 12C2 — Space Agent Installation Readiness

Purpose:
Plan and review Space Agent installation requirements without installing by default. This prompt is for assessment and planning only.

Prerequisites (ALL must be true):
- [ ] Human explicitly approves assessing Space Agent installation
- [ ] Human understands Phase 12C finding: Space Agent is not installed
- [ ] Human has read docs/phase12/PHASE12C_SPACE_AGENT_MANUAL_UI_BRIDGE_REVIEW.md
- [ ] Human has read docs/phase12/AGENT_ZERO_SPACE_AGENT_BRIDGE_BOUNDARIES.md
- [ ] No secrets will be copied into repo

Hard Boundaries:
- Do not install Space Agent by default.
- Do not use sudo unless separately reviewed and approved.
- Do not copy secrets, tokens, or API keys into the repo.
- Do not grant repo write authority.
- Do not grant host write authority.
- Do not enable autonomous tasks.
- Do not run OpenCode implementation prompts.
- Do not modify system/security configs.
- Do not expose Ollama on LAN.

Installation Assessment Steps:
1. Identify official Space Agent source:
   - Check https://github.com/Anthropic/space-agent or official source
   - Verify latest release and Linux compatibility
   - Check if AppImage, Flatpak, or container image available

2. Evaluate installation methods:
   - User-local install (preferred)
   - AppImage (preferred for portability)
   - Flatpak (if available)
   - Container/Podman (if available)
   - System package (avoid unless necessary)

3. Verify system requirements:
   - CPU/RAM requirements
   - Disk space
   - Network requirements
   - Bazzite/Fedora Atomic compatibility

4. Check config requirements:
   - Where does Space Agent store config?
   - Does it require API keys?
   - Does it require external network access?
   - Can it be configured for local-only operation?

5. Security review:
   - Does Space Agent request file system access?
   - Does it request network access?
   - Does it run commands automatically?
   - Can it be sandboxed?

Decision Gates:
- If no official Linux build: BLOCK and document
- If requires sudo: STOP and ask for explicit approval
- If requires external API keys by default: BLOCK or plan fallback
- If cannot be sandboxed: RISK — document and ask for approval

Expected Output:
- Installation feasibility report
- Recommended installation method
- Security assessment
- Config requirements
- Decision: install / block / defer

Note:
This is a future optional prompt. Do not install Space Agent unless explicitly requested after this assessment.
