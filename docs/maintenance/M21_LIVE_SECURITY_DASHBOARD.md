## Summary

M21 builds a local-only live security dashboard for the Bazzite security stack. The dashboard is static HTML generated by a bash script, with an optional foreground watch mode that refreshes at a configurable interval. No daemon, server, or systemd unit is created.

---

## What Was Built

### 1. Security Dashboard Generator Script

**File:** `scripts/gemma-security-dashboard-build`

Features:
- Generates static HTML dashboard, Markdown report, and log file
- `--help` / `-h` flag
- `--open` flag to open in browser
- `--watch` flag for foreground watch mode
- `--interval SECONDS` for watch mode (default 60, minimum 15)
- Watch mode adds `<meta http-equiv="refresh">` to HTML
- Watch mode adds a banner: "WATCH MODE — Refreshing every Xs — Press Ctrl+C in terminal to stop"
- 10 panels: Security Summary, Firewall, USBGuard, ClamAV/Freshclam, Lynis, Security Timers, Recent Reports, Gemma Advisory Status, Guardrails, Next Safe Actions
- Read-only inspection only
- Graceful fallback if tools missing
- Secret redaction

### 2. Operator Guide

**File:** `docs/dashboard/SECURITY_DASHBOARD_OPERATOR_GUIDE.md`

Covers:
- What the dashboard is and is not
- How to generate (one-shot, --open, --watch)
- Panel-by-panel explanation
- Watch mode behavior
- Safety guarantees
- Comparison with main dashboard

## Remaining Uncertainty

1. **firewalld details without sudo:** `firewall-cmd --state` works without sudo on this system, but `firewall-cmd --get-active-zones` may not on all systems. The script gracefully degrades.
2. **Watch mode browser behavior:** The `<meta refresh>` tag should cause the browser to reload, but some browsers may cache the file. Manual refresh (Ctrl+Shift+R) may be needed.
3. **Timer detection reliability:** Timer detection uses `systemctl list-timers | grep`, which is a heuristic. If timer names change, detection will need updating.
4. **freshclam service name:** The script checks both `clamav-freshclam` and `freshclam`. On some systems the service name may differ.
