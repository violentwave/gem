# Memory Query Checklist

**Phase:** 8B.1
**Status:** Concise checklist for supervised memory query execution
**Production status:** RuVector remains prototype-only

---

## Purpose

This checklist is a quick-reference for a future supervised memory query using the Workflow 8B.1 process. It ensures every query is bounded, compared, and human-approved.

---

## Pre-Query Checklist

### Scope and Source Verification
- [ ] Query scope confirmed (approved sources only or explicit scoped path)
- [ ] Source class identified (A/B/C/D per `MEMORY_SOURCE_POLICY.md`)
- [ ] Class D sources (denied) are blocked
- [ ] Class C sources (future approval) flagged for human approval
- [ ] Query does not request broad filesystem indexing

### Denied Data Check
- [ ] No `.env` files in query scope
- [ ] No API keys or provider secrets
- [ ] No raw private logs
- [ ] No browser data (cookies, Local Storage, Session Storage, Trust Tokens)
- [ ] No private project code (unless explicitly scoped by human)
- [ ] No broad roots: `~/projects/`, `~/offload/security-reports/`, `~/.cache/`, `~/.config/`, `~/.local/share/`

### Stage 3A Fallback Verification
- [ ] Stage 3A JSONL index is available at `~/.local/share/bazzite-security/gemma-knowledge/index/chunks.jsonl`
- [ ] Stage 3A is confirmed as canonical fallback
- [ ] Equivalent keyword query prepared for Stage 3A

---

## Execution Checklist

### RuVector Semantic Query (Future Phase Execution)
- [ ] RuVector semantic prototype queried (if available)
- [ ] Embedding model confirmed: `nomic-embed-text:latest` (768 dims)
- [ ] Top semantic matches with relevance scores captured
- [ ] Key excerpts extracted from top results
- [ ] **Skip in Phase 8B.1** — documentation-only

### Stage 3A Deterministic Query (Mandatory)
- [ ] Stage 3A query executed with equivalent keyword wording
- [ ] Top deterministic matches with keyword matches captured
- [ ] Key excerpts extracted from top results
- [ ] Results saved for comparison

### Comparison (Mandatory)
- [ ] Source filename overlap calculated (see note below)
- [ ] Heading/topic overlap compared
- [ ] Agreement classified: strong / partial / weak / contradiction / insufficient evidence
- [ ] Top sources identified from both systems
- [ ] Whether each source is from an approved doc verified
- [ ] Whether Stage 3A found a stronger source noted
- [ ] Whether RuVector found useful context missed by Stage 3A noted
- [ ] Whether results are contradictory noted

> **Note:** Overlap percentage alone is not enough. Source relevance matters more than raw overlap numbers.

### Source Relevance Review
- [ ] Each top source relates to query topic
- [ ] Excerpts contain query-relevant information
- [ ] No denied sources in results
- [ ] Human spot-check performed if weak agreement or contradiction

---

## Post-Query Checklist

### Gemma Synthesis (Only if Explicitly Requested)
- [ ] Human explicitly requested Gemma synthesis
- [ ] Retrieved context provided to Gemma wrapper (not raw query)
- [ ] Synthesis output labeled as advisory-only
- [ ] Uncertainty and limitations noted

### Uncertainty and Fallback
- [ ] Uncertainty sections completed in output
- [ ] If RuVector data missing/stale: noted, fell back to Stage 3A
- [ ] If both systems return insufficient results: query expanded or escalated
- [ ] Fallback path documented in report

### No Ingestion Performed
- [ ] No new sources indexed during query
- [ ] No memory ingestion triggered
- [ ] RuVector remains prototype-only
- [ ] No production memory promotion occurred

### No Autonomous Actions
- [ ] Query only — no fixes, no edits, no changes
- [ ] No autonomous learning loop activated
- [ ] Human approval captured for any follow-up actions

### Report Generation
- [ ] Output formatted using `docs/workflows/templates/memory-query-output-template.md`
- [ ] Report saved to `~/offload/security-reports/manual/` (default)
- [ ] Repo-local report (`reports/`) only if human explicitly requested
- [ ] Query ID, date, scope recorded in metadata

- [ ] RuVector results table included (if available)

- [ ] Sources Used table completed

  - `high_confidence_supported`
  - `supported_with_partial_overlap`
  - `semantic_useful_but_stage3a_stronger`

  - `ruvector_only_needs_review`

  - `blocked_denied_scope`

## Validation Checklist

### Standard Validators (Must All Pass)
```bash
gemma-evals-status    # Must be PASS
gemma-evals-check     # Must be PASS
gemma-examples-check  # Must be PASS
```

### Boundary Verification
- [ ] No secrets in query results
- [ ] No `.env` files accessed
- [ ] No raw logs accessed
- [ ] No browser data accessed
- [ ] No broad filesystem roots indexed
- [ ] No repo edits made
- [ ] No system/package/model config changes made
- [ ] No autonomous Agent Zero or Space Agent tasks spawned

### Human Approval Captured
- [ ] Query scope approved (if Class C sources involved)
- [ ] Gemma synthesis approved (if requested)
- [ ] Contradiction resolution approved (if strong disagreement)
- [ ] Report reviewed by human
- [ ] Follow-up actions approved (if any beyond retrieval)

---

## Stop Conditions Checklist

Verify none of these were triggered (or if triggered, handled correctly):

- [ ] Query requested secrets/private data → blocked
- [ ] Query required broad filesystem indexing → blocked
- [ ] Query asked to ingest new source without approval → routed to ingestion review
- [ ] RuVector and Stage 3A strongly disagreed on critical claims → escalated to human
- [ ] Retrieved sources outside approved paths → discarded, violation reported
- [ ] Semantic prototype data missing or stale → fell back to Stage 3A only
- [ ] Stage 3A fallback unavailable → stopped, escalated to human
- [ ] User asked for production memory promotion → blocked
- [ ] User asked for autonomous action instead of retrieval → blocked

---

## Quick Reference: Quality Labels

| Label | Meaning |
|-------|---------|
| `high_confidence_supported` | Both systems agree, sources approved, high relevance |
| `supported_with_partial_overlap` | Agreement with differences, still useful |
| `semantic_useful_but_stage3a_stronger` | RuVector found context but Stage 3A more authoritative |
| `stage3a_only` | RuVector unavailable or useless; Stage 3A carries answer |
| `ruvector_only_needs_review` | RuVector returned results but Stage 3A did not; human review needed |
| `insufficient_evidence` | Both systems returned < 3 relevant results |
| `blocked_denied_scope` | Query touched denied paths or requested private data |
